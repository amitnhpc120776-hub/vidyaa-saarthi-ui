# Notes for Integration - `aria-describedby` on the trigger must reference the `id` of the associated `vs-tooltip__panel`. - `aria-hidden` on `vs-tooltip__panel` is toggled by JavaScript when visibility changes. - Use the root modifier classes to set placement, trigger behavior, theme, arrow presence, content type, behavior flags, and size. - Apply `.is-visible` to the root to show the panel; remove or use `.is-hidden` to hide while keeping it mounted. - Add `.is-disabled` to the root to ignore interactions. - When using `vs-tooltip--html`, ensure interactive content manages focus appropriately.
